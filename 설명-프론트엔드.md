# Hermes Connection 프론트엔드 UI/UX 설명

이 문서는 Obsidian 플러그인 화면, 상호작용, 렌더링 구조를 설명합니다.

## 1. 화면 구성

채팅 화면은 크게 네 영역입니다.

```text
Header
Toolbar
Chat log
Composer
```

### Header

표시 내용:

- Hermes title
- Remote session client subtitle
- 연결 상태 pill
- Settings icon

연결 상태:

- `Connected`
- `Connecting`
- `Unconnected`

상태 텍스트는 상세 메시지를 title로 가지고 있어 hover 시 원인 확인에 도움이 됩니다.

## 2. Toolbar

현재 toolbar 구조:

```text
[+] [History] [folder icon + project select] [chat icon + session select]
```

### New session

`+` 아이콘 버튼입니다.

동작:

- Hermes에 새 세션 생성 요청
- 새 세션을 현재 세션으로 설정
- 마지막 세션 id 저장

### History

history 아이콘 버튼입니다.

과거 refresh 버튼을 제거하고 같은 위치를 History 관리 버튼으로 재사용했습니다.

동작:

- History modal 열기
- 세션 목록 확인
- 세션 검색
- 세션 선택
- 세션 삭제

### Project select

folder icon이 붙은 select입니다.

목적:

- 전체 세션 또는 Hermes project 범위를 선택
- 선택한 project에 맞게 session select 필터링

UI 세부:

- select 내부에는 실제 option만 들어갑니다.
- 아이콘은 select 바깥 wrapper에 absolute 배치합니다.
- Obsidian/native select option 안에는 SVG/icon을 직접 넣을 수 없으므로 wrapper 방식이 안전합니다.

### Session select

message-square icon이 붙은 select입니다.

목적:

- 현재 project 범위에 해당하는 세션 선택
- 세션 선택 시 Hermes에서 메시지 history 로드

상호작용:

- `pointerdown`
- `focus`
- `change`

드롭다운을 열거나 포커스가 갈 때 `syncSessionsQuietly()`로 최신 세션을 조용히 가져옵니다.

## 3. History modal

History modal은 세션 관리 화면입니다.

주요 UI:

```text
History
Search session title or messages...
[sync icon]

Sessions / Search results
  session title
  snippet / message count / model / cwd
  delete icon
```

### 검색

검색창은 debounce 후 실행됩니다.

우선 순위:

1. Hermes server search API
2. local fallback search

Hermes 검색 API:

```text
GET /api/sessions/search?q=...
```

검색 결과는 `session_id + snippet` 형태일 수 있습니다. 플러그인은 이미 불러온 세션 목록과 매칭해서 제목, 모델, 메시지 수를 보강합니다.

### 세션 선택

검색 결과나 목록 row를 클릭하면:

1. 해당 세션이 속한 project를 찾습니다.
2. project select를 맞춥니다.
3. session select를 맞춥니다.
4. `loadSessionHistory(sessionId)`를 호출합니다.
5. modal을 닫습니다.

### 세션 삭제

delete icon을 누르면 확인창을 띄웁니다.

안전장치:

- 현재 active session은 삭제 불가
- confirm 후 삭제
- 삭제 후 세션 목록 재동기화
- 검색 cache에서 제거

실제 삭제는 백엔드 로직 문서의 `deleteSession`을 참고하세요.

## 4. Chat log

채팅 로그는 `this.lines` 배열을 화면에 렌더링합니다.

line role:

```text
user
assistant
system
reasoning
tool
activity
```

`activity`는 서버에서 직접 오는 role이 아니라 화면 렌더링 단계에서 생성하는 가상 묶음입니다.

## 5. Thinking / Tool / Activity 렌더링

### 기존 문제

Hermes가 긴 작업을 하면 화면이 다음처럼 길어졌습니다.

```text
Thinking
Tool
Tool
Tool
Assistant answer
```

이러면 사용자 질문과 최종 답변 사이가 너무 멀어집니다.

### 현재 방식

렌더링 단계에서 assistant 답변 직전의 연속된 `reasoning/tool` 구간을 하나의 `Activity`로 압축합니다.

예:

```text
Activity 1 thinking · 2 tools
Assistant answer
```

Activity를 펼치면 내부 개별 항목을 볼 수 있습니다.

```text
Activity
  Thinking
  Tool: search_files
  Tool: read_file
```

### 진행 중 애니메이션

진행 중인 `Thinking`에만 shimmer/pulse 효과를 표시합니다.

완료된 상태:

- Activity shimmer 없음
- Thinking shimmer 없음
- dot pulse 없음

이유:

완료된 Activity가 계속 반짝이면 사용자가 아직 작업 중이라고 오해합니다.

구현 포인트:

- 진행 중 reasoning에만 `is-active` class 부여
- CSS에서 `.hermes-remote-reasoning.is-active::before`에만 shimmer 적용
- 기본 `.hermes-remote-thinking-dot`은 animation none
- `.is-active .hermes-remote-thinking-dot`에만 pulse 적용

## 6. Assistant 답변 렌더링

assistant 답변은 Markdown으로 렌더링합니다.

사용 API:

```text
MarkdownRenderer.renderMarkdown
```

fallback:

```text
setText(rawText)
```

답변 카드에는 copy icon이 있습니다.

동작:

- assistant 메시지 hover 시 copy button 표시
- 클릭하면 답변 전체 텍스트를 clipboard에 복사

## 7. User 메시지 표시

사용자 메시지는 오른쪽 bubble 형태입니다.

주의:

플러그인이 Hermes에 보내는 실제 prompt에는 system/context가 붙을 수 있지만, UI에는 사용자가 입력한 원문만 보여줘야 합니다.

history 로드 시에도 `stripObsidianPromptForDisplay()`로 `[User message]` 이후만 표시합니다.

## 8. Mention UI

mention UI는 Copilot/Agent Client 사용감을 참고했습니다.

지원 항목:

```text
Active Note
Notes
Folders
Vault
No Context
```

### Mention menu

동작:

- 입력 중 `@` 감지
- mention button 클릭 시 강제 open
- blur 시 바로 닫히지 않도록 suppress 시간 사용
- 검색창 클릭 시 메뉴가 내려가거나 닫히지 않게 처리

### Active Note chip

기본적으로 현재 active note를 자동 mention처럼 보여줍니다.

중요:

현재 파일이 바뀌면 active note chip도 따라가야 합니다.

이를 위해 workspace 이벤트를 사용합니다.

```text
active-leaf-change
file-open
```

## 9. Composer

하단 입력창은 agent-client/Codex 계열 UI를 참고했습니다.

구성:

```text
textarea
@ mention button
access select
model select
reasoning select
send icon
```

### access select

```text
Safe
Full
```

Read mode는 제거했습니다.

이유:

Hermes live WebSocket에서 현재 확인된 세션 권한 토글은 `yolo`입니다. `read_only` 또는 `access_mode=read_only`에 해당하는 안정적인 세션 API는 확인되지 않았습니다.

따라서 프론트엔드는 실제 Hermes 동작과 맞는 두 단계만 제공합니다.

동작:

- `Safe`를 선택하면 현재 Hermes 세션에 `yolo=off`를 적용합니다. Hermes가 위험 명령으로 분류한 작업은 승인 요청을 보낼 수 있습니다.
- `Full`을 선택하면 현재 Hermes 세션에 `yolo=on`을 적용합니다. 위험 명령 승인 prompt도 우회될 수 있습니다.
- 모드 변경은 저장될 뿐 아니라 현재 열린 세션에도 즉시 반영됩니다.
- 승인/거절 버튼은 Hermes가 `approval.request` 이벤트를 보낼 때 표시됩니다.
- `approval.request`가 와도 채팅 UI는 live WebSocket 수신을 닫지 않습니다. 승인 후 이어지는 tool 로그와 최종 답변을 같은 스트림에서 계속 표시합니다.
- 실시간 이벤트를 놓치는 경우를 대비해 승인 후 세션 기록을 주기적으로 다시 읽는 fallback이 있습니다. 기준 시점 이후 assistant 답변이 발견되면 UI를 갱신하고 polling을 멈춥니다.
- 단순 파일 생성, 복사, 이동, 단일 파일 삭제처럼 Hermes가 위험하지 않다고 판단한 작업은 Safe에서도 승인 없이 실행될 수 있습니다.

### model select

Hermes에서 가져온 모델 목록 중 visible model만 보여줍니다.

모델 목록이 길 때는 설정의 visible model editor에서 표시 대상을 조정합니다.

### reasoning select

```text
Fast
Med
Deep
```

## 10. 모바일/iPad 고려사항

모바일에서 중요한 점:

- toolbar 높이를 낮게 유지
- 버튼은 박스보다 아이콘 중심
- select는 icon + compact text 구조
- composer 높이를 과도하게 키우지 않음
- History modal은 화면 폭의 94vw 이내
- chat log는 `user-select: text`를 유지

```bash
git diff --check
```
