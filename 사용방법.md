# Hermes Connection 사용방법

이 문서는 Obsidian 안에서 Hermes 홈서버와 대화하기 위한 `Hermes Connection` 플러그인 사용 방법을 설명합니다.

`Hermes Connection`은 Obsidian 자체가 모델을 실행하는 플러그인이 아닙니다. Obsidian은 채팅 UI와 노트 문맥 전달을 맡고, 실제 모델 호출, 파일 읽기/수정, 터미널 실행, 승인 처리는 Hermes 서버가 맡습니다.

## 1. 준비물

필요한 것:

- Obsidian
- Hermes가 설치된 서버 컴퓨터
- 같은 네트워크 또는 Tailscale
- Hermes dashboard username/password
- Obsidian Vault
- 이 플러그인 폴더 `hermes-connection`

권장 구조:

```text
iPad / Mac / Windows Obsidian
  -> Hermes Connection 플러그인
  -> Tailscale 또는 내부 네트워크
  -> Mac Studio / 서버 노트북의 Hermes
  -> 모델, 도구, 파일 작업 실행
```

## 2. 플러그인 설치

1. Obsidian Vault 폴더를 엽니다.
2. Vault 안의 `.obsidian/plugins/` 폴더로 이동합니다.
3. `hermes-connection` 폴더를 복사합니다.

예시:

```text
내Vault/
  .obsidian/
    plugins/
      hermes-connection/
        manifest.json
        main.js
        styles.css
```

4. Obsidian을 다시 실행합니다.
5. `Settings -> Community plugins`에서 `Hermes Connection`을 켭니다.

주의:

- `data.json`은 GitHub에 올리면 안 됩니다.
- `data.json`에는 서버 주소, username/password, gateway token, 마지막 세션 정보가 저장될 수 있습니다.
- GitHub 배포용에는 `manifest.json`, `main.js`, `styles.css`만 포함하는 것을 권장합니다.

## 3. Hermes 서버 실행

가장 단순한 원격 사용 방식은 Hermes dashboard 서버를 고정 포트로 여는 것입니다.

서버 컴퓨터에서 실행:

```bash
hermes serve --host 0.0.0.0 --port 9119
```

정상 실행되면 비슷한 메시지가 나옵니다.

```text
HERMES_DASHBOARD_READY port=9119
Hermes Web UI -> http://0.0.0.0:9119
```

같은 컴퓨터에서만 테스트:

```text
http://127.0.0.1:9119/api/status
```

다른 기기에서 접속할 때는 `127.0.0.1`을 쓰면 안 됩니다. `127.0.0.1`은 항상 자기 자신을 뜻합니다. 플러그인은 이제 로컬 자동 감지를 기본으로 사용하지 않고, 사용자가 입력한 `Hermes server URL`만 기준으로 연결합니다.

예시:

```text
Mac에서 접속: http://127.0.0.1:9119
iPad에서 접속: http://서버의_IP주소:9119
Tailscale에서 접속: http://서버의_Tailscale_IP:9119
```

## 4. Dashboard 로그인 설정

Hermes dashboard가 인증을 요구하면 username/password가 필요합니다.

Hermes dashboard 등록은 Hermes 버전에 따라 명령이 다를 수 있습니다. 현재 확인된 방식은 다음 흐름입니다.

```bash
hermes dashboard register
```

화면에서 인증 방식을 고릅니다.

```text
How do you want to authenticate the dashboard?
[1] Username & password
[2] OAuth via Nous Portal
[3] Cancel
```

초보자에게는 `[1] Username & password` 방식이 가장 단순합니다.

플러그인에는 다음 두 값만 입력하면 됩니다.

```text
Dashboard username: 예) admin
Dashboard password: 예) 본인이 설정한 비밀번호
```

과거에는 live gateway token, dashboard provider 같은 값을 따로 입력해야 했지만, 현재 플러그인은 username/password로 자동 로그인한 뒤 WebSocket ticket을 발급받아 연결합니다.

## 5. 같은 Wi-Fi에서 연결하기

서버 컴퓨터의 IP 주소를 확인합니다.

macOS Wi-Fi 예시:

```bash
ipconfig getifaddr en0
```

예를 들어 결과가 다음과 같다면:

```text
172.30.100.19
```

iPad 또는 다른 기기에서 접속 주소는 다음입니다.

```text
http://172.30.100.19:9119
```

플러그인 설정에는 다음처럼 입력합니다.

```text
Hermes server URL: http://172.30.100.19:9119
Dashboard username: admin
Dashboard password: 비밀번호
Use Hermes live gateway: 켬
```

학교, 회사, 카페 Wi-Fi에서는 기기끼리 통신이 막혀 있을 수 있습니다. 이 경우 같은 Wi-Fi여도 iPad에서 Mac으로 접속이 안 될 수 있습니다.

## 6. Tailscale로 외부 접속하기

외부에서 iPad로 집의 Mac Studio나 서버 노트북에 붙으려면 Tailscale을 추천합니다.

### 6.1. 서버 컴퓨터 설정

1. 서버 컴퓨터에 Tailscale 설치
2. 로그인
3. Tailscale IP 확인

macOS 예시:

```bash
tailscale ip -4
```

결과 예시:

```text
100.123.26.117
```

Hermes 서버 실행:

```bash
hermes serve --host 0.0.0.0 --port 9119
```

### 6.2. iPad 설정

1. iPad에 Tailscale 설치
2. 같은 계정으로 로그인
3. Tailscale VPN 활성화
4. Safari에서 테스트

```text
http://100.123.26.117:9119/api/status
```

응답이 뜨면 연결 준비가 된 것입니다.

플러그인 설정:

```text
Hermes server URL: http://100.123.26.117:9119
Dashboard username: admin
Dashboard password: 비밀번호
Use Hermes live gateway: 켬
```

주의:

- `https`가 아니라 `http`를 사용합니다.
- `127.0.0.1`은 iPad에서 쓰면 안 됩니다.
- iPad에서 `127.0.0.1`은 iPad 자신을 뜻합니다.

## 7. 플러그인 설정값 설명

### Hermes server URL

Hermes dashboard 서버 주소입니다.

예시:

```text
http://192.168.0.10:9119
http://100.123.26.117:9119
```

`127.0.0.1`은 Hermes 서버가 실행 중인 바로 그 컴퓨터에서만 테스트용으로 사용할 수 있습니다. iPad/iPhone/Android/다른 Mac/Windows에서 같은 서버를 바라보려면 내부 IP나 Tailscale IP를 입력하세요.

### Dashboard username

Hermes dashboard 로그인 아이디입니다.

예시:

```text
admin
```

### Dashboard password

Hermes dashboard 로그인 비밀번호입니다.

이 값은 플러그인 설정 파일에 저장됩니다. iCloud나 NAS로 Vault를 동기화하면 다른 기기에도 같이 넘어갈 수 있습니다.

혼자 쓰는 개인 환경에서는 편리하지만, Vault를 다른 사람과 공유한다면 주의해야 합니다.

### Use Hermes live gateway

켜는 것을 권장합니다.

이 옵션을 켜면 Hermes의 실시간 이벤트를 받아옵니다.

- 답변 스트리밍
- thinking/reasoning 표시
- 도구 실행 로그
- 승인/거절 요청
- 작업 상태

### Workspace access

Hermes에게 전달하는 권한 의도입니다.

```text
Ask before changes
```

파일 수정, 삭제, 명령 실행 같은 작업은 Hermes 승인 요청을 통해 진행합니다.

```text
Full access
```

서버 정책이 허용하는 범위에서 더 적극적으로 작업합니다. 중요한 Vault에서는 신중하게 사용하세요.

### Reasoning profile

모델의 추론 강도 힌트입니다.

```text
Swift
Balanced
Deep focus
```

간단한 질문은 `Swift`, 복잡한 코드/문서 분석은 `Deep focus`가 적합합니다.

## 8. 채팅 사용 예시

### 8.1. 일반 질문

```text
이 노트 내용을 간단히 요약해줘
```

현재 파일이 자동 문맥으로 붙어 있으면 현재 노트를 기준으로 답합니다.

### 8.2. 현재 파일 질문

```text
@현재파일 이 문서에서 핵심 결론만 뽑아줘
```

또는 멘션 버튼을 눌러 Active Note를 선택합니다.

플러그인은 현재 보고 있는 파일을 문맥으로 전달합니다.

### 8.3. 특정 노트 멘션

```text
@[[프로젝트/회의록.md|회의록]] 여기에서 결정사항만 정리해줘
```

멘션 UI에서 Notes를 선택한 뒤 검색해서 넣을 수도 있습니다.

### 8.4. 현재 폴더 질문

```text
@현재폴더 이 폴더에 어떤 문서들이 있는지 구조를 보여줘
```

현재 보고 있는 노트의 폴더를 기준으로 파일 목록과 일부 파일 내용을 전달합니다.

멘션을 직접 넣지 않아도 다음처럼 자연어에 현재 폴더 표현이 들어가면 플러그인이 자동으로 현재 폴더 문맥을 붙입니다.

```text
현재 폴더에서 오래된 초안 글을 찾아줘
이 폴더에 있는 문서 중 중복된 내용을 정리해줘
지금 폴더에 있는 파일 구조를 보여줘
```

파일 삭제나 수정처럼 위험한 작업은 Hermes 승인 정책에 따라 승인/거절 버튼이 표시될 수 있습니다.

### 8.5. 폴더 멘션

```text
@폴더:프로젝트/2026_오토카 이 폴더의 전체 구조를 요약해줘
```

폴더 안의 Markdown 파일 목록이 전달됩니다. 내용은 너무 많을 수 있으므로 일부만 inline context로 붙습니다.

현재 플러그인은 폴더나 Vault 전체처럼 큰 범위가 선택되면 Hermes에 RAG 검색 힌트를 함께 보냅니다. Hermes에 `obsidian-docsearch` 또는 `docsearch-mcp` 같은 MCP 검색 서버가 연결되어 있으면, 모델은 필요한 경우 검색 도구를 사용해 누락된 파일 내용을 찾을 수 있습니다.

예를 들어 다음 질문은 폴더 목록만 보는 것이 아니라 MCP 검색까지 쓰도록 유도됩니다.

```text
@폴더:프로젝트/2026_오토카 이 폴더 전체에서 네트워크 설정과 카메라 설정이 어떻게 연결되는지 찾아서 정리해줘
```

단순히 파일명 목록만 묻는 경우에는 플러그인이 보낸 폴더 구조만으로 답할 수 있습니다.

```text
@현재폴더 이 폴더에 어떤 파일들이 있어?
```

### 8.6. 파일 수정 요청

```text
@현재파일 문장을 더 자연스럽게 고쳐줘. 원래 의미는 유지해줘.
```

권한 모드가 `Ask before changes`이면 Hermes가 실제 수정 작업 전에 승인 요청을 보낼 수 있습니다. Obsidian 채팅 UI에 승인/거절 버튼이 표시됩니다.

에이전트형 폴더 작업 예시:

```text
현재 폴더에 좀 오래된 글은 찾아서 삭제 후보로 정리해줘
현재 폴더에서 6개월 이상 수정 안 된 임시 메모를 삭제해줘
```

첫 번째처럼 “삭제 후보로 정리”는 보통 읽기/검색 중심입니다.  
두 번째처럼 실제 삭제를 요청하면 Hermes가 파일 목록을 확인하고, 필요하면 approval 요청을 보낸 뒤 사용자가 승인했을 때 삭제를 진행합니다.

### 8.7. 삭제 요청

```text
바탕화면에 있는 테스트 파일 hello.txt를 삭제해줘
```

중요:

- `바탕화면`은 Hermes 서버 컴퓨터의 Desktop을 뜻합니다.
- macOS/Linux에서는 보통 `$HOME/Desktop`
- Windows에서는 사용자 Desktop known folder입니다.
- 플러그인은 Hermes에게 현재 작업 폴더가 아니라 OS의 Desktop 경로를 확인하라고 지시합니다.

## 9. 모델 선택

플러그인은 Hermes에서 제공하는 모델 목록을 가져와 드롭다운으로 보여줍니다.

Hermes 앱에서 Model edit 또는 visible model 설정을 사용하면, 플러그인도 가능한 한 그 설정을 읽어 표시 모델을 맞춥니다.

다만 Hermes가 공식 API로 visible models 양방향 동기화를 제공하지 않는 경우, 플러그인에서 직접 바꾼 토글이 Hermes 앱 설정까지 반영되지는 않을 수 있습니다.

## 10. RAG/MCP 검색 사용

Vault 전체나 큰 폴더를 매번 전부 프롬프트에 넣으면 느리고 비효율적입니다.

권장 방식:

```text
현재 파일 / 작은 노트 -> 플러그인이 직접 내용 전달
큰 폴더 / Vault 전체 -> Hermes MCP 검색 또는 RAG 도구 사용
```

예시 구조:

```text
Obsidian 질문
  -> Hermes Connection이 현재 파일/멘션/폴더 메타데이터 전달
  -> 큰 범위면 ragRecommended 힌트 추가
  -> Hermes가 필요하면 MCP 검색 도구 호출
  -> 검색 결과와 대화 문맥을 합쳐 답변
```

현재 플러그인이 큰 범위에서 함께 보내는 힌트:

```text
ragRecommended: true
ragSearchProvider: obsidian-docsearch/docsearch-mcp
ragSearchScopeType: current-folder | vault-folder | vault-all
ragSearchScopePath: 선택한 Vault 상대 경로
folderFilePaths: 검색 범위 안의 파일 목록
```

Hermes에 연결된 MCP 서버 이름은 환경마다 다를 수 있습니다. 현재 테스트한 구성에서는 Hermes에 다음 이름으로 연결되어 있었습니다.

```text
obsidian-docsearch
```

검색 도구 이름은 Hermes 내부에서 다음처럼 노출되었습니다.

```text
mcp_obsidian_docsearch_doc_search
```

docsearch-mcp 같은 검색 서버를 사용하는 경우, 해당 서버의 watcher가 Vault를 인덱싱하고 있어야 최신 파일이 검색됩니다.

동작 확인 예시:

```bash
hermes mcp list
hermes mcp test obsidian-docsearch
```

정상이라면 `doc-search` 도구가 발견됩니다.

```text
doc-search - Hybrid semantic+keyword search across local files and Confluence
```

주의:

- 플러그인이 직접 docsearch를 호출하는 것은 아닙니다.
- 플러그인은 Hermes에게 검색이 필요한 상황과 검색 범위를 알려줍니다.
- 실제 검색 도구 호출은 Hermes 모델과 도구 시스템이 수행합니다.
- 작은 로컬 모델은 툴콜을 놓칠 수 있으므로, 큰 Vault 검색 테스트는 Codex 계열이나 툴콜이 강한 모델로 확인하는 것이 좋습니다.

## 11. 문제 해결

### 연결이 안 됩니다

확인할 것:

1. Hermes 서버가 실행 중인지
2. `hermes serve --host 0.0.0.0 --port 9119`로 열었는지
3. iPad/다른 기기에서 `/api/status`가 열리는지
4. 주소에 `https`가 아니라 `http`를 썼는지
5. Tailscale VPN이 켜져 있는지
6. 학교/회사 Wi-Fi에서 기기간 통신이 차단된 것은 아닌지

### `127.0.0.1`로는 왜 안 되나요?

`127.0.0.1`은 항상 현재 기기 자신입니다.

Mac에서 `127.0.0.1`은 Mac입니다.  
iPad에서 `127.0.0.1`은 iPad입니다.

iPad에서 Mac의 Hermes에 접속하려면 Mac의 내부 IP 또는 Tailscale IP를 써야 합니다.

### models failed가 뜹니다

가능한 원인:

- username/password가 틀림
- Hermes dashboard 인증이 아직 설정되지 않음
- 서버 URL이 잘못됨
- Hermes 모델 provider가 연결되지 않음
- 너무 많은 로그인 시도로 Hermes가 잠시 429 rate limit을 반환함

잠시 기다린 뒤 다시 Test Connection을 누릅니다.

### thinking이 답변처럼 보입니다

이 플러그인은 Hermes live gateway 이벤트 중 `message.delta`를 답변으로, `reasoning.delta`를 thinking으로 처리합니다.

Hermes 버전이 이벤트 이름을 바꾸면 thinking/answer 분리가 다시 깨질 수 있습니다. 이 경우 `main.js`의 gateway event mapping 부분을 점검해야 합니다.

### 파일이 엉뚱한 위치에 만들어집니다

Hermes 서버의 현재 작업 폴더와 사용자가 말한 위치 표현이 다를 수 있습니다.

예:

```text
바탕화면에 파일 만들어줘
```

이 요청은 서버 OS의 Desktop 폴더로 해석되어야 합니다. 만약 홈 폴더에 만들어진다면 모델이 상대 경로로 잘못 처리한 것입니다.

더 확실한 요청:

```text
서버 컴퓨터의 바탕화면/Desktop 경로를 먼저 확인한 뒤, 그 위치에 test.md를 만들어줘.
```

### 승인 버튼이 안 뜹니다

확인할 것:

- `Use Hermes live gateway`가 켜져 있는지
- 권한 모드가 `Ask before changes`인지
- Hermes 쪽 승인 정책이 활성화되어 있는지
- Hermes 이벤트가 `approval.request` 형태로 오는지

Hermes 이벤트 이름이 바뀌면 플러그인 쪽 mapping 수정이 필요할 수 있습니다.
