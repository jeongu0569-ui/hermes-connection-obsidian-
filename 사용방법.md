# Hermes Connection 사용방법

`Hermes Connection`은 Obsidian 안에서 Hermes 홈서버와 대화하기 위한 원격 채팅 클라이언트입니다.

Obsidian 플러그인은 채팅 UI, 세션 선택, 노트 문맥 전달을 맡습니다. 실제 모델 호출, 파일 읽기/수정, 터미널 실행, 도구 사용, 승인/거절 처리는 Hermes 서버가 담당합니다.

## 1. 전체 구조

```text
Obsidian on Mac/iPad/Windows
  -> Hermes Connection plugin
  -> Hermes dashboard URL
  -> Hermes serve live WebSocket
  -> Model / tools / filesystem / terminal
```

이 플러그인은 기본 기능에서 로컬 `child_process`, stdio ACP, 데스크톱 전용 Electron API에 의존하지 않습니다. 그래서 iPad, iPhone, Android, Windows, macOS를 모두 고려한 구조입니다.

## 2. 설치

Vault 안에 플러그인 폴더를 넣습니다.

```text
내Vault/
  .obsidian/
    plugins/
      hermes-connection/
        manifest.json
        main.js
        styles.css
```

그 다음 Obsidian에서:

1. `Settings -> Community plugins`로 이동합니다.
2. `Hermes Connection`을 켭니다.
3. 오른쪽 패널 또는 ribbon icon에서 `Hermes Connection`을 엽니다.

주의:

- `data.json`은 GitHub에 올리면 안 됩니다.
- `data.json`에는 서버 주소, dashboard username/password, 마지막 세션 정보가 들어갈 수 있습니다.
- 개인용으로 iCloud/NAS/WebDAV에 Vault를 동기화하면 이 설정도 기기간 동기화됩니다.

## 3. Hermes 서버 실행

서버 컴퓨터에서 Hermes dashboard를 고정 포트로 실행합니다.

```bash
hermes serve --host 0.0.0.0 --port 9119
```

정상 실행 예시:

```text
HERMES_DASHBOARD_READY port=9119
Hermes Web UI -> http://0.0.0.0:9119
```

같은 컴퓨터에서만 테스트할 때:

```text
http://127.0.0.1:9119/api/status
```

다른 기기에서 접속할 때는 `127.0.0.1`을 쓰면 안 됩니다. iPad에서 `127.0.0.1`은 Hermes 서버가 아니라 iPad 자기 자신입니다.

다른 기기용 예시:

```text
같은 Wi-Fi: http://서버_IP:9119
Tailscale: http://서버_Tailscale_IP:9119
```

## 4. Dashboard 계정 만들기

Hermes dashboard가 인증을 요구하면 username/password가 필요합니다.

현재 확인된 등록 흐름:

```bash
hermes dashboard register
```

질문이 나오면 초보자에게는 username/password 방식이 가장 단순합니다.

```text
How do you want to authenticate the dashboard?
[1] Username & password
[2] OAuth via Nous Portal
[3] Cancel
```

플러그인에는 username/password만 입력합니다. 플러그인이 자동으로 로그인하고 `/api/ws` WebSocket ticket을 발급받습니다.

## 5. 플러그인 설정

Obsidian `Settings -> Hermes Connection`에서 설정합니다.

일반 사용자는 아래 세 가지만 입력하면 됩니다.

- Hermes server URL
- Dashboard username
- Dashboard password

모델/세션/메시지/승인 endpoint는 플러그인 내부 기본값을 사용합니다. 잘못 수정하면 연결이 깨질 수 있으므로 설정 화면에는 노출하지 않습니다.

### Hermes server URL

Hermes dashboard 주소입니다.

예시:

```text
http://192.168.0.10:9119
http://100.123.26.117:9119
```

Mac에서만 테스트할 때는 `http://127.0.0.1:9119`도 가능하지만, iPad/다른 기기에서는 내부 IP 또는 Tailscale IP를 써야 합니다.

### Dashboard username

Hermes dashboard 로그인 아이디입니다.

예시:

```text
admin
```

### Dashboard password

Hermes dashboard 로그인 비밀번호입니다.

개인용 편의를 위해 플러그인 `data.json`에 저장됩니다. Vault를 다른 사람과 공유한다면 주의하세요.

### Default context scope

Obsidian 문맥을 얼마나 보낼지 정합니다.

```text
No context       -> 노트 문맥을 보내지 않음
Current note     -> 현재 노트 또는 선택 영역을 첨부
Current folder   -> 현재 폴더 구조와 일부 노트 첨부
Vault folder     -> 지정한 Vault 폴더 구조와 일부 노트 첨부
Vault all        -> 전체 Vault 파일 목록과 RAG/search 힌트 중심
```

`Current note`를 선택하면 별도 토글 없이 현재 노트 본문이나 선택 영역이 함께 전달됩니다.

### Live WebSocket

별도 토글은 없습니다. 플러그인은 기본적으로 `hermes serve`가 제공하는 `/api/ws` 실시간 WebSocket 이벤트를 사용합니다.

이 연결이 성공하면 Hermes의 실시간 이벤트가 Obsidian 채팅 UI에 표시됩니다.

- 답변 스트리밍
- Thinking
- Tool 로그
- Approval 요청
- 진행 상태

승인 요청이 와도 WebSocket 수신은 끊지 않습니다. 사용자가 승인하면 Hermes가 이어서 보내는 thinking, tool, final answer 이벤트를 같은 스트림에서 계속 받습니다.

네트워크 지연, 모바일 sleep, Obsidian 백그라운드 전환 등으로 실시간 이벤트가 누락될 수 있으므로, 승인 후에는 세션 기록도 주기적으로 다시 확인합니다. 이 확인은 고정 시간 제한으로 멈추는 방식이 아니라 최종 assistant 답변을 발견하면 멈추는 fallback입니다.

WebSocket 연결이 실패하면 플러그인은 자동으로 REST/SSE 방식으로 폴백합니다.

주의: 여기서 말하는 Live WebSocket은 `hermes serve`의 `/api/ws` endpoint입니다. 터미널 명령어 `hermes gateway`로 관리하는 Telegram/Discord/WhatsApp 같은 메시징 gateway와는 다른 개념입니다.

## 6. Tailscale로 iPad 연결

외부에서 iPad로 집의 Mac Studio나 서버 노트북에 붙으려면 Tailscale을 권장합니다.

서버 컴퓨터:

```bash
tailscale ip -4
hermes serve --host 0.0.0.0 --port 9119
```

예시 IP:

```text
100.123.26.117
```

iPad:

1. Tailscale 앱 설치
2. 같은 계정으로 로그인
3. VPN 활성화
4. Safari에서 테스트

```text
http://100.123.26.117:9119/api/status
```

플러그인 설정:

```text
Hermes server URL: http://100.123.26.117:9119
Dashboard username: admin
Dashboard password: 비밀번호
```

## 7. 채팅 화면 사용법

상단에는 다음 UI가 있습니다.

```text
[+] [History] [Project select] [Session select]
```

### New session

`+` 아이콘을 누르면 새 Hermes 세션을 만듭니다.

### History

History 아이콘을 누르면 세션 관리 창이 열립니다.

할 수 있는 일:

- 전체 세션 보기
- 세션 제목/대화 내용 검색
- 검색 결과에서 세션 열기
- 세션 삭제

삭제는 실제 Hermes 세션 삭제입니다. 현재 열려 있는 active session은 바로 삭제하지 못하게 막혀 있습니다.

### Project select

폴더 아이콘이 붙은 드롭다운입니다.

Hermes 프로젝트 또는 전체 세션 범위를 고릅니다.

```text
All sessions
project A
project B
```

### Session select

채팅 아이콘이 붙은 드롭다운입니다.

선택한 프로젝트에 속한 세션을 고릅니다. 세션을 고르면 해당 Hermes 대화 기록을 불러옵니다.

## 8. 입력창 사용법

하단 입력창에서 메시지를 보냅니다.

```text
Message Hermes - @ to add context
```

Enter:

- 메시지 전송

Shift+Enter:

- 줄바꿈

오른쪽 아래에는 다음 옵션이 있습니다.

```text
Safe | 모델 | 추론모드
```

### Safe / Full

Hermes 실행 guard 모드입니다.

`Safe`:

- 플러그인은 현재 Hermes 세션에 `yolo=off`를 적용합니다.
- Hermes가 위험 명령으로 분류한 작업은 승인 흐름을 거칩니다.
- 예를 들어 `rm -r`, `rm -rf`, `python -c`, `bash -c`, 민감 파일 수정 같은 작업은 승인 요청이 뜰 수 있습니다.
- 단순 `cp`, `touch`, `mkdir`, `mv`, 단일 파일 `rm`처럼 Hermes가 위험하지 않다고 판단한 작업은 승인 없이 실행될 수 있습니다.
- 즉 Safe는 "모든 변경 전 확인" 모드가 아니라 "Hermes 위험 명령 승인 guard 유지" 모드입니다.

`Full`:

- 플러그인은 현재 Hermes 세션에 `yolo=on`을 적용합니다.
- Hermes의 위험 명령 승인 prompt도 우회될 수 있습니다.
- 가능한 변경 작업은 별도 승인 없이 진행될 수 있으므로 신뢰하는 개인 환경에서만 사용하세요.

현재 플러그인에는 Read-only 모드가 없습니다.

- Hermes live WebSocket의 세션 설정에는 현재 `read_only` 또는 `access_mode=read_only`에 해당하는 안정적인 API가 확인되지 않았습니다.
- Read-only를 제대로 지원하려면 Hermes 쪽에서 세션별 toolsets/disabled_toolsets/access_mode API가 추가되어야 합니다.

### 모델 선택

Hermes에서 제공하는 모델 목록을 불러옵니다. 모델 목록은 Hermes dashboard 또는 Hermes 로컬 설정을 기준으로 표시됩니다.

### 추론 모드

```text
Fast
Med
Deep
```

응답 품질과 속도 성향을 정하는 힌트입니다.

## 9. @ 멘션

입력창에서 `@` 버튼을 누르거나 `@`를 입력하면 문맥 선택 메뉴가 열립니다.

지원 항목:

- Active Note
- Notes
- Folders
- Vault
- No Context

### Active Note

현재 보고 있는 노트를 문맥으로 보냅니다.

현재 파일을 바꾸면 active note 문맥도 따라가도록 설계되어 있습니다.

### Notes

Vault 안의 Markdown 노트를 검색해서 멘션합니다.

멘션된 파일은 Obsidian API로 읽어서 Hermes에 전달합니다.

### Folders

폴더를 문맥 범위로 전달합니다.

작은 폴더는 파일 목록과 일부 내용을 inline context로 붙일 수 있습니다. 큰 폴더나 Vault 전체 질문은 RAG/docsearch-mcp 같은 Hermes 도구가 검색하도록 힌트를 전달합니다.

### Vault

Vault 전체 범위를 문맥 metadata로 전달합니다.

전체 Vault 내용을 전부 프롬프트에 붙이는 방식은 피합니다. 대신 Hermes가 검색 도구를 활용할 수 있도록 범위와 RAG 권장 힌트를 보냅니다.

## 10. Thinking / Activity / Tool 표시

Hermes가 답변하는 동안 플러그인은 다음을 보여줍니다.

- Thinking
- Tool 로그
- Activity 묶음
- 최종 답변

최근 UI는 Codex 스타일에 가깝게 정리했습니다.

진행 중:

- Thinking이 얇은 블록으로 표시됩니다.
- 진행 중인 경우에만 은은한 빛 효과가 있습니다.

답변 완료 후:

- 연속된 `Thinking + Tool + Tool` 구간은 `Activity` 하나로 묶입니다.
- 예: `Activity 1 thinking · 2 tools`
- 접혀 있으므로 답변과 사용자 메시지 사이가 과하게 벌어지지 않습니다.
- 펼치면 내부 thinking/tool 상세 로그를 볼 수 있습니다.
- 완료된 Activity에는 반짝임 효과가 남지 않습니다.

## 11. 승인/거절

Hermes가 파일 삭제, 수정, 터미널 명령 등 승인이 필요한 작업을 요청하면 채팅 UI에 승인/거절 버튼이 표시됩니다.

```text
Approve
Reject
```

승인하면 Hermes 서버가 작업을 계속합니다. 거절하면 Hermes 서버에 거절 결과를 돌려줍니다.

승인 후에는 채팅창이 같은 live WebSocket 스트림을 계속 듣습니다. 따라서 Hermes가 승인 이후 실행하는 tool 로그와 최종 답변이 실시간으로 표시됩니다. 만약 실시간 이벤트가 누락되면 플러그인이 세션 기록을 주기적으로 다시 읽어 최종 답변을 보정 표시합니다.

참고:

- 승인 버튼은 모델이 텍스트로 "진행할까요?"라고 물을 때 뜨는 것이 아닙니다.
- Hermes의 실제 도구 실행 guard가 `approval.request` 이벤트를 보낼 때 표시됩니다.
- 한 번 거절된 동일 작업은 Hermes 안전장치가 재시도를 막을 수 있습니다. 이 경우 새 요청이나 새 세션에서 다시 지시하는 편이 안전합니다.

## 12. docsearch-mcp / RAG와의 관계

이 플러그인은 docsearch-mcp를 직접 실행하지 않습니다.

역할 분담:

```text
Obsidian plugin
  -> 현재 질문과 Obsidian 문맥 metadata 전달
  -> 큰 폴더/Vault 질문이면 RAG recommended 힌트 전달

Hermes server
  -> docsearch-mcp 같은 MCP 도구 판단 및 호출
  -> 검색 결과를 바탕으로 답변
```

즉 docsearch-mcp를 Hermes에 연결해두면, 큰 폴더나 Vault 전체 질문에서 Hermes가 검색 도구를 활용하기 쉬워집니다.

## 13. 문제 해결

### iPad에서 접속이 안 될 때

확인할 것:

- Hermes가 `--host 0.0.0.0`으로 실행 중인지
- iPad가 같은 Wi-Fi 또는 Tailscale에 연결되어 있는지
- URL이 `127.0.0.1`이 아닌지
- `http://서버주소:9119/api/status`가 Safari에서 열리는지

### 학교/회사 Wi-Fi에서 안 될 때

공용 Wi-Fi는 기기 간 통신을 막는 경우가 많습니다.

이때는 Tailscale을 쓰는 것이 가장 안정적입니다.

### 세션이 안 보일 때

- History 버튼을 열어 세션 목록을 확인합니다.
- Project select가 특정 프로젝트로 제한되어 있으면 `All sessions`로 바꿉니다.
- Hermes 앱에서 만든 세션이 프로젝트에 묶여 있지 않을 수 있습니다. 이 경우 `All sessions`에서 보입니다.

### 모델 목록이 이상할 때

- Hermes dashboard의 모델 설정을 먼저 확인합니다.
- Ollama/LM Studio/Nous/OpenAI Codex 등 provider 연결 상태를 확인합니다.
- 플러그인 설정의 visible models를 다시 동기화하거나 Obsidian을 재시작합니다.

### 변경사항이 화면에 바로 안 보일 때

Obsidian은 플러그인 JS를 메모리에 올려두기 때문에 `main.js`를 바꾼 뒤에는 Obsidian을 재시작해야 합니다.

CSS만 바뀐 경우도 캐시 때문에 재시작이 가장 확실합니다.
